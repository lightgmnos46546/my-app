# คู่มือเปิดใช้ระบบล็อกอิน + กันข้อมูลชนกัน + Backup (201 SOMS)

ระบบออกแบบให้ **เปิดทีละขั้นโดยไม่ทำให้ของเดิมพัง** — frontend ตัวใหม่ใช้กับ GAS ตัวเก่าได้
และ GAS ตัวใหม่ก็ใช้กับ frontend ตัวเก่าได้ ตราบใดที่ยังไม่เปิด `REQUIRE_AUTH=true`

---

## ขั้นที่ 1 — อัปเดตโค้ด Google Apps Script

1. เปิด Google Sheets หลักของระบบ → เมนู **ส่วนขยาย (Extensions) → Apps Script**
2. **สำรองโค้ดเดิมก่อน**: copy โค้ดเก่าทั้งหมดไปเก็บไว้ (เช่น วางในไฟล์ใหม่ชื่อ `Old.gs.bak`
   แล้วลบ/คอมเมนต์ฟังก์ชัน `doGet` กับ `doPost` เดิมออก เพื่อไม่ให้ชื่อชนกัน)
3. วางโค้ดจากไฟล์ [`Code.gs`](Code.gs) ทั้งหมดลงไป
4. ⚠️ ตรวจตัวแปร `DASHBOARD_SHEETS` บนหัวไฟล์ ให้ตรงกับชีทที่หน้า Dashboard ใช้จริง
   (ค่าเริ่มต้นตรงกับ frontend ปัจจุบันแล้ว)
5. กด **Deploy → Manage deployments → ไอคอนดินสอ (Edit) → Version: New version → Deploy**

   > ❗ ต้องใช้วิธี *แก้ deployment เดิม* เท่านั้น ถ้ากด New deployment ตัวใหม่
   > URL จะเปลี่ยน แล้วต้องไปแก้ `GAS_URL` ใน `src/squadron-dashboard.tsx` ตาม

6. ทดสอบว่าของเดิมยังทำงาน: เปิดเว็บแอปตามปกติ ทุกแท็บต้องโหลดข้อมูลได้เหมือนเดิม

ถึงจุดนี้ได้ของฟรีทันที 2 อย่างโดยยังไม่ต้องเปิดล็อกอิน:
- **AUDIT LOG** — ทุกการบันทึกถูกจดไว้ในชีท `AUDIT LOG` (ตอนนี้เป็น anonymous ไปก่อน)
- **กันข้อมูลชนกัน** — ถ้าสองคนแก้ชีทเดียวกันพร้อมกัน คนที่กดทีหลังจะได้รับแจ้งเตือน
  "มีคนอื่นแก้ข้อมูลไปก่อนหน้า" แทนที่จะเขียนทับงานกันเงียบ ๆ

---

## ขั้นที่ 2 — สร้างชีท USERS (บัญชีผู้ใช้)

สร้างแท็บใหม่ในสเปรดชีตหลัก ชื่อ **`USERS`** (ตัวพิมพ์ใหญ่) หัวคอลัมน์แถวแรก:

| USERNAME | PIN | ROLE | DISPLAY NAME | ACTIVE |
|---|---|---|---|---|
| nat | 123456 | admin | น.ท. ณัฐ | TRUE |
| korn | 234567 | editor | ร.อ. กร | TRUE |
| ops-view | 111111 | viewer | จอ Ops Room | TRUE |

- **ROLE**: `admin` (ทำได้ทุกอย่าง + สั่ง backup) · `editor` (อ่าน+บันทึก) · `viewer` (อ่านอย่างเดียว)
- **PIN**: พิมพ์เป็นเลขธรรมดาได้เลย — ระบบจะ **แปลงเป็น hash อัตโนมัติ** เมื่อผู้ใช้
  ล็อกอินสำเร็จครั้งแรก (จะเห็นในเซลล์เปลี่ยนเป็น `sha256:...` เอง)
- **ACTIVE**: ใส่ `FALSE` เพื่อระงับบัญชีชั่วคราว
- เปลี่ยน PIN ให้ผู้ใช้: ลบค่าในช่อง PIN แล้วพิมพ์ PIN ใหม่เป็นเลขธรรมดา
- ใส่ PIN ผิดเกิน 5 ครั้ง → บัญชีล็อกอัตโนมัติ 10 นาที

---

## ขั้นที่ 3 — ตั้ง Backup รายวันอัตโนมัติ

1. ใน Apps Script → ไอคอน **นาฬิกา (Triggers)** เมนูซ้าย → **+ Add Trigger**
2. ตั้งค่า: function = `dailyBackup` · event source = **Time-driven** ·
   type = **Day timer** · เวลา = **2am–3am**
3. กด Save (อาจมีหน้าต่างขอ permission ครั้งแรก → Allow)

ผลลัพธ์: ทุกคืนระบบจะ copy สเปรดชีตทั้งไฟล์ไปไว้ในโฟลเดอร์ Drive ชื่อ
**"201 SOMS Backups"** และเก็บย้อนหลัง 14 วัน (เก่ากว่านั้นลบให้เอง)
รวมถึงล้าง token หมดอายุ + ตัด AUDIT LOG ไม่ให้เกิน 5,000 แถว

ต้องการ backup ทันทีตอนไหน: รันฟังก์ชัน `dailyBackup` จากตัวแก้ไข Apps Script ได้เลย

---

## ขั้นที่ 4 — เปิดบังคับล็อกอิน (ทำหลัง deploy frontend ใหม่แล้วเท่านั้น)

1. ตรวจว่า frontend เวอร์ชันใหม่ deploy ขึ้น GitHub Pages แล้ว (`npm run deploy`)
2. ใน Apps Script → ⚙️ **Project Settings → Script Properties → Add script property**
   - Property: `REQUIRE_AUTH` → Value: `true`
3. ไม่ต้อง deploy ซ้ำ — มีผลทันที
4. เปิดเว็บ → ต้องเจอหน้าล็อกอิน → ลองล็อกอินด้วยบัญชีในชีท USERS
5. ทดสอบสิทธิ์: บัญชี `viewer` ต้องแก้ข้อมูลไม่ได้ (ระบบจะแจ้งเตือนและฝั่ง server ก็ปฏิเสธซ้ำอีกชั้น)

**ปิดระบบล็อกอินชั่วคราว** (เช่น มีปัญหา): เปลี่ยน `REQUIRE_AUTH` เป็น `false`
→ ระบบกลับไปเปิดแบบเดิมทันที โดย audit log และกันข้อมูลชนกันยังทำงานต่อ

Script Properties เพิ่มเติม (ไม่บังคับ):
- `SESSION_HOURS` = จำนวนชั่วโมงก่อน session หมดอายุ (ค่าเริ่มต้น 24)

## ขั้นที่ 5 — เปิดใช้ปุ่ม "จัดเวร" (แก้สเปรดชีตเวรจากแอป)

ปุ่ม ✏️ จัดเวร ในแท็บ PILOT ON DUTY จะเขียนข้อมูลลง **สเปรดชีตเวรไฟล์เดิม**
(ไฟล์เดียวกับที่แอปอ่านอยู่) แบบเจาะจงเซลล์ — แก้เฉพาะคอลัมน์ ALERT / SOF /
BASE OPS. / EMER BRIEF / REMARK / 9923 (K-M) / C SQDN (Q-T) ของแถววันที่นั้น
คอลัมน์อื่นและการจัดรูปแบบของชีทไม่ถูกแตะเลย

เงื่อนไขที่ต้องครบ:
1. วางโค้ด `Code.gs` เวอร์ชันล่าสุด (มีฟังก์ชัน `handleUpdateDuty_`) + deploy แบบ Edit deployment เดิม
2. **บัญชี Google เจ้าของสคริปต์ ต้องมีสิทธิ์ "แก้ไข" สเปรดชีตเวร**
   (ถ้าเป็นเจ้าของไฟล์อยู่แล้วก็ผ่านเลย) — ครั้งแรกที่ใช้อาจมีหน้าขอ permission เพิ่ม → Allow
3. ถ้าสเปรดชีตเวรเปลี่ยนไฟล์ในอนาคต: ตั้ง Script Property `DUTY_SHEET_ID` เป็น ID ไฟล์ใหม่
   (ตอนนี้ใช้ ID เดิมที่ฝังไว้ในโค้ดเป็นค่าเริ่มต้น)

ข้อจำกัด: จัดเวรได้เฉพาะ **วันที่ที่มีแถวอยู่แล้ว** ในชีทเวร (ระบบไม่เพิ่มแถวใหม่เอง
เพื่อไม่ให้ผิดรูปแบบที่หน่วยใช้) — ถ้าวันที่ยังไม่มี ให้เพิ่มแถวในชีทก่อน
เมื่อเปิดบังคับล็อกอินแล้ว สิทธิ์ `viewer` จะมองไม่เห็นปุ่มจัดเวร และฝั่ง server ก็ปฏิเสธซ้ำอีกชั้น

หมายเหตุ: หลังบันทึก แอปจะอัปเดตหน้าจอทันที แต่ปุ่ม "↻ รีโหลด" อาจเห็นค่าเก่าได้ชั่วครู่
เพราะ Google cache ตัว CSV ไว้สั้นๆ — ค่าจริงในชีทบันทึกเรียบร้อยแล้ว

---

## สิ่งที่ควรรู้

- **ตารางเวร (PILOT ON DUTY)** ดึงจากสเปรดชีตอีกไฟล์แบบ public CSV — ยังเปิดอ่านสาธารณะ
  เหมือนเดิม ไม่อยู่ใต้ระบบล็อกอินนี้ (เป็น read-only อยู่แล้ว)
- ระบบนี้กันคนทั่วไปและกันความผิดพลาดได้ดี แต่ไม่ใช่ระบบความปลอดภัยระดับ classified —
  ข้อมูลลับจริงไม่ควรอยู่บน Google Sheets ที่แชร์แบบ public
- ดูประวัติว่าใครแก้อะไรเมื่อไหร่: เปิดชีท **AUDIT LOG** ในสเปรดชีตหลัก
